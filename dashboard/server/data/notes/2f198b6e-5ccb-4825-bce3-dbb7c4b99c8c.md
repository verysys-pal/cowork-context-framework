### 3. 사용하기

- Multica 설치 이후
    - Multica 데몬은 서버의 `PATH`에 등록된 `opencode` 실행 파일을 찾아 에이전트로 등록합니다.
- 브라우저 접속
    - [http://localhost:3000](http://localhost:3000/)
- 데몬 재시작
    - 설정을 수정한 후, 서버 터미널에서 Multica 데몬을 재시작하여 `opencode`를 새로 스캔
        
        ```bash
        multica daemon stop
        multica daemon start
        ```
        
- Multica에서 OpenCode 에이전트 등록
    - Create Agent 클릭
        
        ![image.png](attachment:f5ae9fa2-14e0-41b0-acad-e6c480a9dcef:image.png)
        
- Agent Instrucitons : 에이전트의 '페르소나'와 '작동 규칙'을 정의
    
    ```bash
    # Role
    당신은 포항 4세대 방사광 가속기(4GSR) 프로젝트를 돕는 전문 엔지니어이자, Python/C++ 코딩 전문가입니다. 특히 EPICS 제어 시스템과 진공 시스템 데이터 처리에 능숙합니다.
    
    # Working Style
    - 코드 수정 시에는 항상 기존 스타일을 준수하고, 변경 사유를 명확히 주석으로 남깁니다.
    - 가급적 작고 집중된 단위로 코드를 수정합니다.
    - 복잡한 로직은 'Deep Thinking' 모드를 활용하여 논리적인 검증을 거칩니다.
    
    # Technical Context
    - 로컬 엔진: OpenCode (Gemma 4:31b 기반)
    - 주요 프로젝트: 4GSR Beamline Control
    - 환경: Rocky Linux 9.6, L40S GPU Server
    ```
    

- 작업할당
    1. **New Issue** (또는 단축키 `C`)를 누릅니다.
    2. **내용 입력**:
        - **Title**:
            - `.bashrc` 파일의 EPICS 경로 오류 수정
        - **Description**:
            - "현재 로그인 시마다 EpicsHostArch 파일을 찾을 수 없다는 에러가 발생합니다. `/home/mhdev/.bashrc` 파일을 검토해서 잘못된 경로를 실제 존재하는 `/usr/local/epics` 하위 경로로 수정해 주세요."
    3. **Assignee**: 방금 설정한 `opencode_GPT_Gemma4` 에이전트를 선택.
