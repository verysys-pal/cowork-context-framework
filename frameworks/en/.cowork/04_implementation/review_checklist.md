# Review Checklist

> Quality checks for AI-generated code and deliverables during Human review

---

## Purpose

When a Human reviews code and documents generated by AI,
this checklist provides a **consistent and thorough verification path**.

---

## 1. Code Review

### Correctness
- [ ] Does it satisfy the requirements (User Story / Acceptance Criteria)?
- [ ] Are edge cases handled?
- [ ] Is error handling appropriate?

### Design
- [ ] Does it align with the architecture and design principles?
- [ ] Is there unnecessary complexity?
- [ ] Is the abstraction level appropriate?

### Code Quality
- [ ] Does it follow `coding_convention.md`?
- [ ] Is duplicate code avoided?
- [ ] Is naming clear?

### Testing
- [ ] Are unit tests written?
- [ ] Do the tests cover meaningful scenarios?
- [ ] Do all tests pass?

### Security
- [ ] Are sensitive values kept out of hardcoded source?
- [ ] Is input validation appropriate?

---

## 2. Document Review

- [ ] Accuracy: is the content factually correct?
- [ ] Completeness: is anything important missing?
- [ ] Consistency: does it avoid contradiction with other documents?
- [ ] Clarity: will the AI understand it in the next session?

---

## 3. Design Review

- [ ] Is traceability to requirements preserved?
- [ ] Are trade-offs recorded explicitly?
- [ ] Are scalability and maintainability considered?
