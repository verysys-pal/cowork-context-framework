## Purpose

This document turns the requirements specification (WHAT) into **concrete behavior flows (HOW)**,
providing a shared baseline of expected behavior for developers and QA.

---

## Writing Rules

- In this document, use visible status values such as `TBD`, `Not written`, `None`, and `Undecided` instead of HTML-comment placeholders.
- If the real feature does not exist yet, do not expand empty structure unnecessarily; keep only the minimum sections.
- Once a feature is confirmed, repeat the same structure by `FUNC-*` unit and append it.
