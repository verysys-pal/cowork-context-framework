## Placeholder 운용 기준

- `인터페이스 이름`, `Provider`, `Consumer`, `프로토콜`은 승인된 실제 값으로 교체한다.
- 엔드포인트/메서드 블록은 실제로 존재하는 계약만 남기고, 사용하지 않는 예시 블록은 삭제한다.
- `Request`와 `Response` 코드 블록에는 합의된 필드 구조 또는 예시 payload를 채우고, 주석만 남겨두지 않는다.
- 메시지/이벤트가 없는 인터페이스는 해당 표를 비워두기보다 `해당 없음`으로 명시하거나 섹션을 제거한다.
- 제약 값이 아직 없으면 빈칸 대신 `미확정`으로 적고, 같은 항목을 `Open Questions`에 등록한다.
- HTML 주석형 placeholder는 남기지 않고 가시 상태값으로 정리한다.
