# Member Profile

> 팀원 프로필 — 이 파일은 세션 최초 시작 시 자동 생성된다

---

## Template 사용 규칙

- 이 파일은 `members/<이름>/profile.md` 생성용 템플릿이다.
- `<!-- -->` 주석은 실제 프로필 작성 후 제거한다.
- 역할과 담당 영역은 실제 배정 기준으로 적고, 미정이면 `미정`으로 명시한다.
- 폴더명과 `이름` 필드는 최초 1회 Human이 확인한 사용자 식별자를 같은 표기로 맞춘다.
- 1인 프로젝트에서는 `권한 = Master`, `역할 = 프로젝트 오너`, `담당 영역 = 프로젝트 전반`을 solo default로 시작하고, 이후 실제 표현으로 좁혀도 된다.

## 개인 작업 공간

프로필 생성 시 아래 구조를 기본으로 사용한다.
1인 프로젝트도 팀 프로젝트와 같은 `workspace/session_logs/` 구조를 처음부터 사용하고, 운영만 더 단순하게 가져간다.

```text
members/<이름>/
├── profile.md              ← 이 파일
├── proposals/              ← Change Proposal 보관
└── workspace/              ← 개인 작업 공간
    ├── my_state.md         ← 개인 상태 인덱스 (다음 AI가 우선 읽음)
    └── session_logs/       ← 개인 세션 로그 (1인/팀 공통, git 제외)
```
