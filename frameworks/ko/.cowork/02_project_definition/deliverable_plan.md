# 산출물 계획 (Deliverable Plan)

> 프로젝트별 공식 산출물 범위 확정과 미수집 데이터 관리 문서

---

## 목적

모든 프로젝트에 기본 추천 14종 전체가 필요한 것은 아니다.
이 문서는 프로젝트 성격에 따라 **실제 생성할 산출물 범위**를 확정하고,
각 산출물에 필요한 데이터를 **의도적으로 수집**하기 위한 계획을 관리한다.

- Define 단계 초반에 AI가 프로젝트를 프로파일링하여 산출물 필요 여부를 제안한다.
- Human이 승인하면 확정되며, 이후 AI의 질문과 데이터 수집이 이 계획을 기반으로 진행된다.
- Intent가 Pivot / Split으로 크게 변경되었을 때 재검토한다.

---

## 운영 경계

- 이 문서는 **무엇을 생성할지**와 **무엇이 아직 부족한지**를 관리한다.
- `핵심 기준 문서` 열은 프로젝트 계획용 메모이며, 실제 기준 문서 해석 우선순위와 생성 방식은 `07_delivery/export_spec.md`를 따른다.
- 기본 추천 14종과 승인된 확장 산출물(15+)의 활성 여부, 승인 시점, 재검토 트리거는 여기서 확정한다.

| 항목 | 내용 |
|------|------|
| 프로젝트 유형 | 라이브러리 / 서비스 / CLI / 데스크톱 앱 / 모바일 앱 / 임베디드 / 기타 |

| 운영 환경 | 로컬 / 클라우드 / 온프레미스 / 하이브리드 |

### 필요 여부 기준

- `필수` — 프로젝트 완료 시 반드시 생성한다.
- `권장` — 프로젝트 품질을 위해 권장하지만 생략 가능하다.
- `해당없음` — 이 프로젝트에 해당하지 않는다.

## 확장 산출물 등록 (15+)

기본 추천 14종으로 충분하지 않을 때만 아래 섹션을 사용한다.

| # | 산출물 | 제안 주체 | 제안 근거 | 필요 여부 | 확정 시점 | 마지막 승인 | 재검토 트리거 | 핵심 기준 문서 | 수집 상태 | 비고 |
|---|--------|----------|----------|----------|----------|-------------|----------------|----------------|----------|------|
| 15+ | | AI / Human | | 필수 / 권장 / 해당없음 | | YYYY-MM-DD / Human | | | | |

- 확장 산출물은 기본 추천 14종 뒤 번호(`15+`)로 추가한다.
- `제안 주체 = AI`는 AI가 필요성을 먼저 제안했다는 뜻이며, Human 승인 전까지 확정이 아니다.
- `마지막 승인`은 기본적으로 `YYYY-MM-DD / Human` 형식을 사용한다.
- `재검토 트리거`에는 Intent 변경, 범위 확대, 배포 대상 변경처럼 이 계획을 다시 열어야 하는 조건만 짧게 적는다.
- 새 기준 본문 또는 새 기준 문서 경로가 필요하면 `01_cowork_protocol/document_change_impact_matrix.md`를 먼저 확인한다.

## 미수집 데이터 목록

| 산출물 | 필요 데이터 | 수집 방법 | 담당 | 우선순위 | 상태 |
|--------|-----------|----------|------|---------|------|
| | | 질문 / 분석 / 코드 역추출 / 외부 참조 | Human / AI | High / Medium / Low | 미수집 / 수집중 / 완료 |

> 목록 문서 + 상세 문서 구조를 사용하는 항목은 인덱스 문서와 상세 문서를 함께 참조한다.

### 수집 상태 판단 기준

- `미수집` — 핵심 기준 문서가 아직 비어 있거나 근거가 없다.
- `수집중` — 일부 데이터는 확보했지만 공식 산출물 생성 기준으로는 부족하다.
- `완료` — 현재 산출물을 생성하기에 필요한 최소 기준 문서가 갖춰졌다.
