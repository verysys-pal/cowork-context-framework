# Review Checklist

> AI 생성 코드 및 산출물의 품질 검증 — Human 리뷰 시 확인 항목

---

## 목적

AI가 생성한 코드와 문서를 Human이 리뷰할 때,
**일관되고 빠짐없는 검증**을 위한 체크리스트를 제공한다.

---

## 1. 코드 리뷰

### 정확성
- [ ] 요구사항(User Story / Acceptance Criteria)을 충족하는가?
- [ ] 엣지 케이스가 처리되었는가?
- [ ] 에러 핸들링이 적절한가?

### 설계
- [ ] 아키텍처/설계 원칙에 부합하는가?
- [ ] 불필요한 복잡성이 없는가?
- [ ] 적절한 추상화 수준인가?

### 코드 품질
- [ ] `coding_convention.md`를 준수하는가?
- [ ] 중복 코드가 없는가?
- [ ] 명명이 명확한가?

### 테스트
- [ ] 단위 테스트가 작성되었는가?
- [ ] 테스트가 의미 있는 시나리오를 커버하는가?
- [ ] 모든 테스트가 통과하는가?

### 보안
- [ ] 민감 정보가 하드코딩되지 않았는가?
- [ ] 입력 검증이 적절한가?

---

## 2. 문서 리뷰

- [ ] 정확성 — 내용이 사실에 부합하는가?
- [ ] 완전성 — 빠진 항목이 없는가?
- [ ] 일관성 — 다른 문서와 모순이 없는가?
- [ ] 명확성 — AI가 다음 세션에서 이해할 수 있는가?

---

## 3. 설계 리뷰

- [ ] 요구사항과의 추적성이 확보되었는가?
- [ ] Trade-off가 명시적으로 기록되었는가?
- [ ] 확장성/유지보수성이 고려되었는가?
