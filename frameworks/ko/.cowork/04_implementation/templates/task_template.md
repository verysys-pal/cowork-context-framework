# Task Template

> 작업 분해 명세 — Plan → Approve → Execute 사이클의 단위

---

## Template 사용 규칙

- 이 파일은 복사 원본이다. 실제 Task 문서를 만들 때는 `TASK-000`, `INT-000`, `MS-000`, `US-000`, `ADR-000`을 실제 ID로 교체한다.
- 연결되지 않은 관련 문서는 placeholder를 남기지 말고 `없음` 또는 해당 없음으로 정리한다.
- 작업 계획은 승인된 실제 단계만 남기고, 사용하지 않는 Step 행은 삭제한다.
- `<!-- -->` 가이드 주석은 초안 작성 후 제거하고, 실행 로그에는 실제 기록만 남긴다.
- `담당`, `상태`, `마지막 갱신일`, `다음 액션`은 `task_registry.md`와 `project_state.md`의 활성 Task 요약과 같은 값으로 유지한다.

## 관련 문서
| 항목 | 참조 |
|------|------|
| 관련 Intent | `INT-000` |
| 관련 Milestone | `MS-000` |
| 관련 User Story | `US-000` |
| 관련 ADR | `ADR-000` (해당 시) |

---

## 동기화 필드

| 항목 | 값 |
|------|----|
| 담당 | Human / AI / Role-* / (Role-* 인수자) |
| 상태 | Planned / In Progress / Review / Done |
| 마지막 갱신일 | YYYY-MM-DD |
| 다음 액션 | 한 줄 요약 또는 없음 |

> 이 4개 필드는 registry와 `project_state.md`의 활성 Task 요약에서 동일한 의미로 유지한다.
