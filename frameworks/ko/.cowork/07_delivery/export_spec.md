# 공식 산출물 생성 규격 (Export Specification)

> 공식 산출물 생성 규칙과 기준 문서 매핑 기준 문서

---

## 목적

`.cowork/`는 **작업 과정의 공유 기준 문서**이고,
`docs/`는 **공식 납품 산출물**이다.

이 문서는 프로젝트 완료 또는 Phase 완료 시,
cowork 기준 문서를 기반으로 공식 문서를 생성하기 위한 **기준 문서 해석 규칙**과 **생성 방식**을 정의한다.

---

## 운영 경계

- 이 문서는 **어떤 기준 문서를 읽고 어떻게 생성할지**를 정의한다.
- 실제 생성 대상의 활성 여부, 승인 상태, 미수집 데이터는 `02_project_definition/deliverable_plan.md`를 따른다.
- 최종 공식 산출물 생성 대상은 `deliverable_plan.md`의 활성화된 기본 추천 14종과 Human 승인된 확장 산출물(15+)의 합이다.

---

## 공식 산출물 생성 철학 (Export Philosophy)

이 프레임워크의 공식 산출물 생성은 **고정 포맷 복제**보다 **추적 가능하고 더 나은 공식 문서 생성**을 목표로 한다.

### 불변 규칙

- 공식 산출물은 대응되는 cowork 기준 문서와 추적 가능해야 한다.
- 필수 섹션, 핵심 결정, 주요 제약, 미해결 위험을 누락하지 않는다.
- 기준 문서가 부족한 항목은 숨기지 않고 `미작성` 또는 `미확정`으로 명시한다.
- AI가 추론을 사용한 경우, 확정 사실과 혼합하지 않고 `가정`, `추론`, `추가 확인 필요` 형태로 구분한다.

### AI 재량 허용 범위

- AI는 독자의 이해를 위해 섹션 순서, 표 구성, 설명 수준을 더 나은 형태로 재구성할 수 있다.
- 중복되는 내용을 통합하거나, 여러 기준 문서의 연결 관계를 더 명확하게 설명할 수 있다.
- AI 성능이 향상될수록 더 완성도 높은 합성과 설명을 시도할 수 있다.
- 단, 위 불변 규칙과 기준 문서 추적성을 깨는 재구성은 허용하지 않는다.

## 기준 문서 해석 우선순위 (Source Priority)

공식 산출물 생성 시 기준 문서 해석 우선순위는 아래와 같다.

1. 승인된 기준 본문
2. 목록 문서 + 상세 문서 조합
3. 승인된 ADR
4. 최신 세션 로그
5. imported context

- `templates/*_template.md`는 공식 산출물 생성 기준 문서가 아니다.
- `session_logs/`, `imported_context/`는 상위 기준 문서가 비어 있을 때 보조 근거로만 사용한다.
- imported context의 내용은 가능한 한 먼저 기준 본문, 목록 문서, 상세 문서로 승격한 뒤 공식 산출물 생성에 사용한다.

## 확장 산출물(15+) 생성 해석

- 확장 산출물은 `deliverable_plan.md`의 `확장 산출물 등록 (15+)`에 있고 Human 승인된 항목만 공식 산출물 생성 대상에 포함한다.
- 확장 산출물의 번호, 제목, 핵심 데이터 기준 문서는 `deliverable_plan.md`를 기준으로 확정한다.
- 단일 기준 본문이면 `직접 생성`, 복수 기준 문서면 `합성`, 목록 문서 + 상세 문서 구조면 해당 조합 규칙을 따른다.
- 기본 추천 14종 중 하나에 흡수 가능한 경우에는 별도 공식 문서를 늘리지 않고 기존 문서에 통합하는 쪽을 우선 검토한다.

## 생성 방식 정의

### 직접 생성

cowork 기준 문서 파일의 내용을 공식 문서 형식으로 정리하여 `docs/`에 출력한다.

- 템플릿 가이드 주석(`<!-- -->`)을 제거한다.
- 문서 번호, 버전, 승인 정보 헤더를 추가한다.
- 필요 시 목차를 생성한다.
- 섹션 순서 재배치, 중복 제거, 표현 개선은 가능하다.
- 사용한 기준 문서를 문서 헤더 또는 참조 섹션에 명시한다.
- 확정되지 않은 내용은 `가정` 또는 `추가 확인 필요`로 분리한다.

### 합성

여러 cowork 기준 문서를 하나의 공식 문서로 통합한다.

- 여러 기준 문서를 엮을 때는 어느 문서에서 어떤 내용을 가져왔는지 추적 가능하게 남긴다.
- AI는 더 나은 설명 구조를 제안할 수 있지만, 근거 없는 세부사항을 확정 사실처럼 추가하지 않는다.

### 목록 문서 + 상세 문서 처리 규칙 (Registry + Instance)

- 목록 문서는 목록, 상태, 우선순위, 링크의 기준선으로 사용한다.
- 상세 문서는 상세 설명, 결정 근거, 개별 작업 계획을 보완하는 기준 문서로 사용한다.
- WBS 공식 산출물 생성은 `milestone_registry.md`와 `task_registry.md`를 기본 골격으로 삼고, 세부 설명이 필요할 때만 `tasks/TASK-*.md`를 펼친다.
- 아키텍처 공식 산출물 생성은 `adr_registry.md`와 `adrs/ADR-*.md`를 함께 읽어 승인된 결정만 본문에 반영한다.

## 공식 산출물 생성 시점

| 시점 | 대상 | 트리거 |
|------|------|--------|
| Phase 완료 시 | 해당 Phase 관련 산출물 | Quality Gate 통과 후 |
| 프로젝트 완료 시 | 활성화된 기본 추천 14종 + 승인된 확장 산출물(15+) | Human `릴리즈` 선언 |
| 수시 | 개별 문서 | Human 요청 시 |
